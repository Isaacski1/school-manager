# Demo Video Recording Guide

Use this guide privately when recording videos for the public Watch Demos page. Do not copy these instructions into the live site.

## Recording Rules

- Use a clean demo school with sample students, parents, teachers, fees, and reports.
- Hide real phone numbers, parent names, API keys, payment secrets, and private school data.
- Keep each video focused. Do not explain every menu item.
- Record at 1080p if possible.
- Use a calm voiceover and move slowly enough for viewers to follow.
- Before recording, prepare the page and sample data so you do not spend time typing long forms.
- End each video with the benefit, not with setup details.

## Recommended Recording Order

1. Platform Overview
2. Admin Dashboard Overview
3. Fees & Payments
4. Parent Portal Overview
5. Student & Parent Records
6. Attendance Tracking
7. Assessments & Report Cards
8. SMS & Parent Communication
9. Teacher Portal Overview

## 1. Platform Overview

Target length: 3-4 minutes  
Audience: School owners and decision makers  
Filename: `platform-overview.mp4`

Record these areas:

- Marketing home page or landing page.
- Login screen.
- School Admin dashboard.
- Main sidebar modules.
- Brief explanation of Admin, Teacher, Parent, and Super Admin roles.
- Quick jump through Students, Fees, Attendance, Reports, and Communication.

Suggested voiceover:

“School Manager GH brings daily school operations into one platform. Administrators can manage students, fees, attendance, academic reports, and parent communication, while teachers and parents access the information they need from their own portals.”

## 2. Admin Dashboard Overview

Target length: 4-5 minutes  
Audience: School administrators  
Filename: `admin-dashboard.mp4`

Record these areas:

- Dashboard summary cards.
- Sidebar navigation.
- Student count, fee/payment overview, attendance summary, and activity areas.
- Quick visit to Students, Teachers, Fees & Payments, Attendance, Report Cards, and SMS Reminders.
- Show how the admin dashboard acts as the control center.

Suggested voiceover:

“The admin dashboard gives school leaders a quick picture of what is happening across the school. From here, they can move into students, fees, attendance, reports, payroll, communication, and other daily operations.”

## 3. Student & Parent Records

Target length: 4-5 minutes  
Audience: Admins and records officers  
Filename: `student-parent-records.mp4`

Record these areas:

- Student list.
- Add student or open an existing student profile.
- Student details, class, and academic information.
- Father, mother, guardian, or parent contact fields.
- Explain that these records support fees, reports, attendance, and SMS communication.

Suggested voiceover:

“Every student profile keeps the key details schools need, including class information and parent contacts. These records connect to fees, reports, attendance, and parent communication.”

## 4. Attendance Tracking

Target length: 3-4 minutes  
Audience: Admins and teachers  
Filename: `attendance-tracking.mp4`

Record these areas:

- Attendance module.
- Select class or student list.
- Mark present, absent, or late.
- Save attendance.
- Show attendance history or dashboard attendance status.

Suggested voiceover:

“Attendance tracking helps schools record daily presence quickly and review attendance history whenever needed.”

## 5. SMS & Parent Communication

Target length: 4-5 minutes  
Audience: Admins  
Filename: `sms-parent-communication.mp4`

Record these areas:

- SMS wallet balance.
- Top-up button or SMS credit area.
- Parent recipient list.
- Message composer and templates.
- Send SMS confirmation.
- Sent broadcast logs.
- Parent dashboard notice section if you want to show in-app communication.

Suggested voiceover:

“Schools can send important reminders to parents using SMS, and they can also send in-app notices to parent dashboards. This helps keep parents informed about fees, reports, meetings, and school updates.”

## 6. Fees & Payments

Target length: 5-7 minutes  
Audience: Admins, bursars, and school owners  
Filename: `fees-payments.mp4`

Record these areas:

- Fees & Payments page.
- Fee setup or assigned fees.
- Select a student.
- Record a payment.
- Show balance update.
- Show payment history.
- Generate or preview receipt/invoice if available.
- Show online payment tracking if relevant.

Suggested voiceover:

“The Fees & Payments module helps schools track what parents owe, record payments, update balances, and keep payment history organized.”

## 7. Assessments & Report Cards

Target length: 5-8 minutes  
Audience: Admins and teachers  
Filename: `assessments-report-cards.mp4`

Record these areas:

- Assessment or academic report area.
- Select class, term, and subject.
- Enter marks.
- Calculate totals, grades, or positions.
- Generate or preview report cards.
- Show how the report can be shared or printed if available.

Suggested voiceover:

“Schools can manage academic records, enter marks, calculate results, and prepare report cards from one place.”

## 8. Teacher Portal Overview

Target length: 3-5 minutes  
Audience: Teachers  
Filename: `teacher-portal.mp4`

Record these areas:

- Teacher login or teacher dashboard.
- Assigned classes.
- Attendance workflow.
- Marks or report entry workflow.
- Any teacher-specific notices or tools.

Suggested voiceover:

“Teachers can focus on the classes and academic work assigned to them, including attendance and assessment tasks.”

## 9. Parent Portal Overview

Target length: 3-4 minutes  
Audience: Parents and school owners  
Filename: `parent-portal.mp4`

Record these areas:

- Parent login or parent dashboard.
- Child profile or student summary.
- Fees and payment status.
- Notices or reminders.
- Report cards or academic updates.

Suggested voiceover:

“Parents can view important updates about their child, including fees, school notices, and academic reports.”

## Upload Checklist

After recording and editing, export the videos using these filenames:

- `platform-overview.mp4`
- `admin-dashboard.mp4`
- `student-parent-records.mp4`
- `attendance-tracking.mp4`
- `sms-parent-communication.mp4`
- `fees-payments.mp4`
- `assessments-report-cards.mp4`
- `teacher-portal.mp4`
- `parent-portal.mp4`

Place the files in:

```text
public/videos/
```

The Watch Demos page expects those filenames.
